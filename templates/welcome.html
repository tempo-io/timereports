Getting started


{{appName | i18n}} for Jira expands Jira's built-in time tracking function and includes the following Reports:

  • Time Entry - Provides a convenient way to record hours worked in real-time (for instance, start, stop, resume timer, log work done, or edit previous records).
  • Time Tracker - Provides the most extensive way to analyze user workloads. It displays a classic sheet view of hours worked per day and includes many options to customize the report.
  • Calendar - Provides a summary of hours worked per day in a calendar view.
  • Timespent - Provides a simplified way to analyze user workload. It displays a classic sheet view with only the Time Spent column and includes many options to customize the report. Use the Pivot by Field option to extend Time Spent by the field value when the worklog starts.
  • Pivot by User - Displays a summary of hours worked for a project during a specific period. The columns contain project users (resources), and the rows contain project issues (tasks).
  • Pivot by Status - Displays the time worked on issues arranged by the issue statuses. This report spreads hours worked across all statuses for the issue, starting from the Started Time. You can only use Started Time by enabling Started Time In Status in the User Default Settings.
  • Pivot by Label - Displays a summary of hours worked for a project spread over columns from worklog comments.
  • Time Tracking - Displays Estimated Time versus actual Time Spent per issue.
  • Pie Chart - Displays a visual summary of the time worked on a project in a certain period.
  • Time in Status - Displays the time elapsed for particular issues. This report calculates the time passed between status changes based on the issue change history. You can filter the columns with the Time In Status Categories option in the User Default Settings.
  • Resolution Time - Displays the time elapsed for resolved and unresolved issues.
  • Time Balance - Displays all changes in the time tracking fields for issues in a certain period. Note: There is no access to worklogs from this report page.
  • Cost Report - Displays costs for a certain period based on the hourly rates.

Note: These are the same reports found in Time Tracker Lite. Time Tracker offers additional features listed here.

To access the reports, navigate to Projects > your project > Reports > Other > Time Tracker. To record hours worked from the issue in real-time, use the Time Entry Issue Panel options for groups and projects in Configuration. Note: Depending on your permissions, you can select the Time Tracker icon using the Apps or Project menu. The Apps menu icon is available if you are in the Auditors Group or the Time Entry Issue Panel Group. The Project menu icon is available if you are only on Time Entry Issue Panel Projects. You can also add a quick link with the Jira Application Navigator, which lets you set a link that is viewable only by groups you specify.

Additional features of {{appName | i18n}}

Time Tracker for Jira Cloud includes the same reports as Time Tracker Lite and also includes these additional features:

  • Log work for other users - Select specific groups to be able to log time impersonating other users.
  • Email subscription - You can send an email or create an email subscription for your preferred reports.
  • REST API integration: You can use REST API integrations in Time Tracker for Jira Cloud.
  • Log Suggested Work Automatically: Enable the Log Suggested Work option in User Preferences to log suggested work automatically.
